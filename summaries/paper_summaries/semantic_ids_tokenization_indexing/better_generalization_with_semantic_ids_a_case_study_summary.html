Better Generalization With Semantic Ids: A Case Study In Ranking For Recommendations

← К общему списку статей

Авторы: Anima Singh, Trung Vu, Nikhil Mehta, Raghunandan Keshavan, Maheswaran Sathiamoorthy, Yilin Zheng, Lichan Hong, Lukasz Heldt, Li Wei, Devansh Tandon, Ed H. Chi, Xinyang Yi.

Аффилиации: Google DeepMind; Google / YouTube.

Индустрия: YouTube production ranking.

Первичный источник: arXiv source 2306.08121.

Коротко

  • Uses Semantic IDs to replace random hashed video IDs in ranking.
  • RQ-VAE learns 8-code SID from frozen 2048D multimodal content embeddings.
  • SentencePiece over SID sequences outperforms manual n-grams at large table sizes.

Контекст

  • Random ID embeddings memorize but do not generalize.
  • Content embeddings generalize but can lose memorization capacity.
  • Semantic collisions should be better than random collisions.

Проблема

  • Need compact representation for billions of videos and watch history features.
  • Direct dense input degraded quality in YouTube ranking.
  • Hashing trick causes meaningless collisions.

Метод/архитектура

  • Stage 1: train and freeze RQ-VAE.
  • Stage 2: ranking model consumes SID subpieces.
  • N-gram SID uses unigram/bigram chunks.
  • SPM-SID learns variable-length subpieces from SID corpus.

Objective/алгоритм

  • Ranking model still optimizes CTR objective.
  • RQ-VAE uses residual quantization over content embeddings.
  • SPM vocabulary groups frequent SID fragments, analogously to NLP subword tokenization.

Ablation conclusions

  • Without user history, some n-gram SID can underperform random hashing because usage skew matters.
  • With user history, SID representations improve because history covers diverse content.
  • SPM better than hand-crafted n-grams at larger capacities.

Сильные стороны

  • Strong production ranking perspective.
  • Separates memorization/generalization trade-off.
  • Shows semantic IDs as feature representation, not only retrieval target.

Ограничения

  • Data and absolute metrics are private.
  • Requires strong multimodal video encoder.
  • Frozen tokenizer may still face long-term drift.

Как реализовать/проверять

  1. Зафиксировать версию каталога, train/eval split и mapping item/document -> identifier; без этого невозможно понять, улучшает ли метод саму модель или только меняет пространство кандидатов.
  2. Считать отдельно качество tokenization и качество generator/ranker: collision rate, utilization, Gini, valid-path rate, Recall/HR/NDCG/MRR и latency должны лежать в одном отчете.
  3. Делать ablation не только по среднему качеству, но и по head/torso/tail, cold-start, new users/new items, long-history и категориям с похожими объектами.
  4. Проверять деградацию при обновлении каталога: semantic IDs могут устаревать, а генератор может помнить старые пути.
  5. Сохранять обратный индекс identifier -> item/document и явно логировать случаи many-to-one collisions.
  6. Для генеративного вывода использовать constrained decoding или post-validation, иначе invalid identifiers будут маскироваться в offline метриках.
  7. В production считать стоимость не только модели, но и перестроения codebook, trie/index, feature pipeline и мониторинга drift.
  8. В отчетах отделять retrieval-stage gains от downstream ranking/business metrics, потому что рост HR@K не всегда дает CTR/CVR uplift.

Failure modes и мониторинг

  • Identifier collapse: малая часть кодов получает большую долю объектов, а генератор начинает переиспользовать популярные пути.
  • Semantic-collaborative mismatch: похожие по тексту объекты имеют разные user intents, или наоборот, совместно потребляемые объекты текстово далеки.
  • Exposure bias autoregressive generation: ошибка раннего токена полностью меняет candidate path.
  • Popularity bias: модель учится генерировать frequent IDs и выглядит хорошо на aggregate, но теряет novelty и tail coverage.
  • Evaluation leakage: если ID/tokenizer обучался на будущем каталоге или post-training видел target-like сигналы, offline gain завышен.
  • Serving mismatch: offline beam шире и медленнее production beam, поэтому качество не переносится напрямую.

Итог

  • Semantic IDs create meaningful collisions.
  • SentencePiece adaptation is a practical way to inject SID into ranking models.
