UniGRec: Unified Generative Recommendation with Soft Identifiers for End-to-End Optimization

Авторы: Jialei Li, Yang Zhang, Yimeng Bai, Shuai Zhu, Ziqi Xue, Xiaoyan Zhao, Dingxian Wang, Frank Yang, Andrew Rabinovich, Xiangnan He.

Аффилиации: University of Science and Technology of China; National University of Singapore; The Chinese University of Hong Kong; Upwork.

1. Коротко: о чем статья

UniGRec предлагает принципиально иной подход к обучению генеративных рекомендательных систем на основе semantic ID. Вместо двухстадийной схемы, где сначала обучается токенизатор (RQ-VAE), а потом замораживается при обучении рекомендательной модели, UniGRec делает весь pipeline дифференцируемым через soft identifiers -- непрерывные распределения по codewords, через которые проходят градиенты от recommendation loss обратно в токенизатор.

Авторы выявляют три фундаментальные проблемы такого end-to-end подхода: (1) расхождение между мягким обучением и жестким inference, (2) коллапс codeword'ов из-за неравномерного использования, (3) дефицит collaborative signal при обучении только на уровне отдельных токенов. Для каждой проблемы предлагается отдельный механизм: Annealed Inference Alignment (AIA), Codeword Uniformity Regularization (CUR) и Dual Collaborative Distillation (DCD) от легковесной teacher-модели.

На трех датасетах (Upwork, Amazon Beauty, Amazon Pet) UniGRec превосходит все baselines, включая TIGER, LETTER, EAGER, OneRec, ETEG-Rec и DiscRec-T.

2. Контекст и мотивация

Генеративная рекомендация состоит из двух ключевых компонентов: токенизатора, который преобразует item'ы в последовательности дискретных кодов из codebook'ов, и рекомендательной модели (обычно autoregressive Transformer), которая по истории пользователя генерирует коды следующего item'а. Это позволяет избежать исчерпывающего перебора кандидатов и повышает масштабируемость.

Существующие парадигмы обучения такой системы имеют серьезные ограничения. В staged training токенизатор обучается первым и замораживается -- его идентификаторы "агностичны к downstream recommendation objective". В alternating optimization токенизатор и рекомендательная модель обновляются поочередно, но остаются асинхронными и не объединены единым objective. UniGRec предлагает третий путь: true end-to-end joint training, где recommendation loss напрямую влияет на параметры токенизатора через дифференцируемые soft identifiers.

3. Проблема: три failure mode end-to-end обучения

Авторы формулируют три ключевых вызова, которые возникают при переходе к end-to-end обучению с soft identifiers.

Training-Inference Discrepancy. Во время обучения каждый item представлен непрерывным распределением по codewords, что обеспечивает gradient flow. Но на inference нужен дискретный жесткий код для lookup'а в таблице item'ов. Если обучение проводилось с мягкими распределениями, а inference работает с argmax, возникает систематический разрыв между train и serve.

Item Identifier Collapse. End-to-end оптимизация может сконцентрировать использование на небольшом числе "удобных" codewords. Если модель научилась хорошо предсказывать несколько популярных кодов, остальные codewords деградируют, снижая емкость token space и увеличивая коллизии.

Collaborative Signal Deficiency. Обучение на уровне отдельных токенов фокусируется на fine-grained token prediction, но ослабляет item-level collaborative signals -- информацию о том, какие item'ы покупаются/просматриваются вместе, которая критична для рекомендаций.

4. Базовая механика: RQ-VAE токенизация

UniGRec строится поверх стандартного RQ-VAE framework'а. Для каждого item'а с semantic embedding \(\mathbf{z}\) токенизатор строит последовательность из \(L\) кодов:

\[ [c_1, \ldots, c_L] = \mathcal{T}(\mathbf{z}) \]

MLP-encoder проецирует embedding в latent space:

\[ \mathbf{r} = \text{Encoder}_\mathcal{T}(\mathbf{z}) \]

На каждом уровне \(l\) residual quantization ищет ближайший code vector в codebook \(\mathcal{C}_l = \{\mathbf{e}_l^k\}_{k=1}^K\):

\[ c_l = \arg\min_k \|\mathbf{v}_l - \mathbf{e}_l^k\|^2, \qquad \mathbf{v}_l = \mathbf{v}_{l-1} - \mathbf{e}_{l-1}^{c_{l-1}} \]

где \(\mathbf{v}_1 = \mathbf{r}\). Агрегированное дискретное представление: \(\tilde{\mathbf{r}} = \sum_{l=1}^L \mathbf{e}_l^{c_l}\). Decoder восстанавливает embedding: \(\tilde{\mathbf{z}} = \text{Decoder}_\mathcal{T}(\tilde{\mathbf{r}})\).

Стандартный loss токенизатора:

\[ \mathcal{L}_\text{Token} = \|\tilde{\mathbf{z}} - \mathbf{z}\|^2 + \sum_l \left[\|\text{sg}(\mathbf{v}_l) - \mathbf{e}_l^{c_l}\|^2 + \beta \|\mathbf{v}_l - \text{sg}(\mathbf{e}_l^{c_l})\|^2\right] \]

где \(\text{sg}(\cdot)\) -- stop-gradient, \(\beta = 0.25\).

Взаимодействия пользователя преобразуются в плоскую последовательность токенов \(X = [c_1^1, \ldots, c_L^1, \ldots, c_1^T, \ldots, c_L^T]\), целевой item -- в \(Y = [c_1^{T+1}, \ldots, c_L^{T+1}]\). Рекомендательная модель (T5-based encoder-decoder Transformer) обучается autoregressive generation с loss'ом:

5. Soft Identifiers и Annealed Inference Alignment

5.1. Soft Identifiers

Ключевая идея UniGRec -- замена недифференцируемого argmin на softmax по отрицательным квадратам расстояний:

\[ p_l(k) = \frac{\exp(-\|\mathbf{r}_{l-1} - \mathbf{e}_l^k\|^2 / \tau)}{\sum_j \exp(-\|\mathbf{r}_{l-1} - \mathbf{e}_l^j\|^2 / \tau)} \]

где \(\tau\) -- температура. Soft identifier подается в рекомендательную модель через взвешенную агрегацию embeddings:

\[ \mathbf{o}_l^t = \text{scatter}(\mathbf{0}, \{p_l(k)\}_{k=1}^K) \cdot \mathcal{O} \]

где \(\mathcal{O} \in \mathbb{R}^{|\mathcal{V}| \times D}\) -- таблица embeddings. Функция scatter разбрасывает \(K\)-мерный вектор вероятностей в \(|\mathcal{V}|\)-мерный разреженный вектор, который затем умножается на таблицу embeddings. Это создает полностью дифференцируемый путь от recommendation loss до параметров токенизатора.

5.2. Temperature Annealing

Температура линейно снижается в процессе обучения:

\[ \tau(\text{step}) = \tau_\text{max} - \frac{\text{step}}{\text{total\_step}}(\tau_\text{max} - \tau_\text{min}) \]

По мере снижения \(\tau\) мягкие распределения становятся все более острыми, приближаясь к поведению жесткого argmax на inference. Это критически важный механизм: он позволяет в начале обучения иметь широкие распределения для хорошего gradient flow, а к концу -- практически дискретные коды, совместимые с inference.

6. Codeword Uniformity Regularization

Для предотвращения коллапса codebook вводится регуляризатор разнообразия, штрафующий низкую энтропию усредненных по batch'у распределений:

\[ \mathcal{L}_\text{CU} = \sum_{l=1}^L \sum_{k=1}^K \bar{p}_l(k) \log(\bar{p}_l(k)) \]

где \(\bar{p}_l(k)\) -- средняя вероятность выбора codeword \(k\) на уровне \(l\) по всему batch'у. Минимизация этого выражения эквивалентна минимизации KL-дивергенции к равномерному распределению. Это заставляет модель использовать все codewords примерно одинаково, предотвращая ситуацию, когда несколько популярных кодов захватывают весь token space.

7. Dual Collaborative Distillation

Для компенсации дефицита collaborative signal используется предобученная SASRec модель в качестве teacher'а. Distillation происходит в двух направлениях.

7.1. Distillation для токенизатора

Из encoder'а рекомендательной модели извлекается представление \(\mathbf{h}^\text{Enc}\) (average pooling), из teacher'а -- embedding целевого item'а \(\mathbf{h}^\text{Tea}\). Оба проецируются через линейный слой и подаются в токенизатор, получая распределения \(P_l^\text{Enc}\) и \(P_l^\text{Tea}\). Symmetric KL divergence loss:

\[ \mathcal{L}_\text{CD}^\mathcal{T} = \sum_{l=1}^L \left[D_\text{KL}(P_l^\text{Enc} \| P_l^\text{Tea}) + D_\text{KL}(P_l^\text{Tea} \| P_l^\text{Enc})\right] \]

7.2. Distillation для рекомендательной модели

Из decoder'а извлекается представление \(\mathbf{h}_i^\text{Dec}\) (first-token pooling), из teacher'а -- \(\mathbf{h}_i^\text{Tea}\). Bidirectional InfoNCE loss:

\[ \mathcal{L}_\text{InfoNCE}(A,B) = -\sum_i \log\frac{\exp(\text{sim}(\mathbf{a}_i, \mathbf{b}_i)/\tau')}{\sum_j \exp(\text{sim}(\mathbf{a}_i, \mathbf{b}_j)/\tau')} \]
\[ \mathcal{L}_\text{CD}^\mathcal{R} = \mathcal{L}_\text{InfoNCE}(\{\mathbf{h}_i^\text{Dec}\}, \{\mathbf{h}_i^\text{Tea}\}) + \mathcal{L}_\text{InfoNCE}(\{\mathbf{h}_i^\text{Tea}\}, \{\mathbf{h}_i^\text{Dec}\}) \]

Таким образом, teacher-модель SASRec передает collaborative signal (знание о co-occurrence паттернах) и в токенизатор, и в рекомендательную модель одновременно.

8. Training Pipeline

Обучение происходит в две стадии.

Stage 1: Pretraining токенизатора. Loss:

\[ \mathcal{L}_\text{Pre} = \mathcal{L}_\text{Recon} + \lambda_\text{CU} \cdot \mathcal{L}_\text{CU} \]

Quantization loss не нужен, потому что soft identifiers обеспечивают дифференцируемый supervision. Temperature annealing применяется от \(\tau_\text{max}\) до \(\tau_\text{min}\).

Stage 2: End-to-End Joint Training. Loss:

\[ \mathcal{L}_\text{Joint} = \mathcal{L}_\text{Rec} + \lambda_\text{Recon} \cdot \mathcal{L}_\text{Recon} + \lambda_\text{CD}^\mathcal{T} \cdot \mathcal{L}_\text{CD}^\mathcal{T} + \lambda_\text{CD}^\mathcal{R} \cdot \mathcal{L}_\text{CD}^\mathcal{R} \]

Температура фиксируется на \(\tau_\text{min}\). \(\mathcal{L}_\text{CU}\) исключается, так как распределения уже достаточно острые после pretraining'а.

8.1. Пошаговый алгоритм UniGRec

  1. Pretrain teacher. Сначала обучается SASRec или аналогичная collaborative model, которая будет источником item-level interaction signal для DCD.
  2. Pretrain soft tokenizer. RQ-VAE заменяет argmin на softmax over distances; temperature annealing постепенно переводит soft distributions к почти hard identifiers.
  3. Применить CUR. На Stage 1 batch-average codeword probabilities регуляризуются к uniform distribution, чтобы не допустить early identifier collapse.
  4. Собрать soft history sequences. Для каждого item в user history вместо одного codeword используется probability distribution по codebook level, преобразованная в embedding через scatter/matrix lookup.
  5. Запустить joint training. Recommendation loss обновляет T5 recommender и tokenizer через soft identifier path; reconstruction loss удерживает semantic grounding.
  6. Добавить DCD в обе стороны. Tokenizer distributions выравниваются с teacher-derived collaborative distributions через symmetric KL, а decoder representations выравниваются с teacher embeddings через bidirectional InfoNCE.
  7. Финализировать hard-ID inference. После обучения soft distributions заменяются argmax IDs, строится constrained beam search over hard catalog SIDs, и отдельно проверяется train-inference agreement.

9. Рисунки и что в них важно

Рисунок сравнения training paradigm'ов служит полезной картой всего ландшафта: staged training (TIGER), alternating optimization (ETEG-Rec) и true end-to-end (UniGRec). Он фиксирует, что UniGRec единственный создает прямой градиентный путь от recommendation loss в токенизатор.

Framework figure показывает, как soft identifiers формируют мост между токенизатором и рекомендательной моделью. Каждый item на каждом уровне codebook представлен распределением вероятностей по codewords, а не одним жестким кодом.

PCA-визуализация codebook embeddings критически важна: UniGRec достигает "widely dispersed, isotropic distribution", тогда как TIGER показывает кластеризацию вокруг нескольких доминирующих codewords. Это прямое визуальное подтверждение того, что CUR работает.

Графики collision rate при разных temperature schedule'ах показывают компромисс: слишком быстрое снижение температуры ведет к раннему коллапсу, слишком медленное -- к mismatch'у на inference. Анnealing дает промежуточное поведение, начиная с collision rate'а как у Fixed-High и заканчивая ниже Fixed-Low.

11. Анализ hyperparameter'ов

11.1. Temperature Annealing

Сравниваются три стратегии: Fixed-High (\(\tau=0.01\)), Fixed-Low (\(\tau=0.001\)) и annealed (\(0.01 \to 0.001\)). Annealed strategy начинает с collision rate, похожим на Fixed-High, но затем снижается ниже Fixed-Low. Это обеспечивает и раннюю exploration (разнообразие кодов), и минимальные финальные коллизии.

11.2. Влияние \(\lambda_\text{CU}\)

При сканировании \(\lambda_\text{CU} \in [0, 10^{-5}, \ldots, 10^{-1}]\) обнаруживается сильная отрицательная корреляция между collision rate и accuracy. Пик performance достигается при \(\lambda_\text{CU} = 10^{-4}\). Избыточная регуляризация (\(>10^{-2}\)) приводит к росту коллизий и падению качества -- код становится слишком равномерным, теряя семантическую группировку.

11.3. Codebook Analysis

PCA-визуализация показывает, что UniGRec достигает изотропного распределения codeword embeddings. Энтропия использования codewords у UniGRec существенно выше, чем у TIGER, указывая на почти равномерное использование. TIGER же концентрируется на нескольких доминирующих codewords.

11.4. Identifier Analysis

Сравнение жестких идентификаторов после Stage 1 и Stage 2 показывает: change rate уменьшается с глубиной уровня (coarse-to-fine refinement), более 93% item'ов меняют максимум один уровень. Это означает "recommendation-aware fine-tuning" идентификаторов, а не деструктивное переназначение.

12. Детали реализации

Рекомендательная модель построена на T5 backbone. Item embeddings: 384-dim из sentence-t5-base для Beauty/Pet; online model embeddings для Upwork. Токенизатор: RQ-VAE с encoder dims \(\{512, 256, 128, 64\}\), 3 codebook layers, 256 codewords каждый, dim 32. К каждому item добавляется unique token для предотвращения неразрешимых коллизий.

Stage 1: batch size 1024, lr \(10^{-3}\), \(\tau_\text{max} \in \{0.01, 0.05\}\), \(\tau_\text{min} = 0.001\), \(\lambda_\text{CU} = 10^{-4}\). Stage 2: AdamW, weight decay 0.05, batch size 512, T5 lr \(\in \{2\cdot10^{-3}, 5\cdot10^{-3}, 8\cdot10^{-3}\}\), tokenizer lr \(\in \{2\cdot10^{-7}, 2\cdot10^{-8}\}\), \(\lambda_\text{Recon} = 0.5\), \(\lambda_\text{CD}^\mathcal{T}\) и \(\lambda_\text{CD}^\mathcal{R}\) перебираются в \(\{0.01, 0.05, 0.1, 0.5, 1.0\}\), \(\tau' = 0.07\).

13. Сильные стороны

Цельная end-to-end формулировка. UniGRec -- одна из немногих работ, где recommendation loss напрямую оптимизирует токенизатор. Это концептуально правильно: если tokenizer обучается отдельно от downstream задачи, его оптимум может не совпадать с оптимумом рекомендательной системы.

Явные механизмы против трех failure modes. Каждая из трех проблем (mismatch, collapse, signal deficiency) получает отдельное решение с четким ablation'ом, подтверждающим его вклад. Это делает работу не просто инженерным trick'ом, а систематическим исследованием.

Широкая база baseline'ов. Сравнение с 11 baselines, включая и традиционные (SASRec, BERT4Rec), и генеративные (TIGER, LETTER, EAGER, OneRec, ETEG-Rec, DiscRec-T), дает уверенность в результатах.

Production-like данные Upwork. Включение production dataset'а (employer-freelancer hiring) расширяет проверку за пределы стандартных Amazon benchmarks. Это особенно ценно для оценки практической применимости.

Teacher distillation без тяжелого LLM. Использование легковесного SASRec в качестве teacher'а добавляет collaborative signal без требования дорогого LLM-backbone.

14. Слабые стороны, ограничения и критика

Сложность обучения. Soft identifier training дороже и сложнее staged tokenizer'а. Две стадии обучения с разными learning rate для tokenizer и T5, temperature annealing schedule, четыре hyperparameter'а для весов loss'ов -- все это увеличивает engineering overhead. В production среде такая сложность может быть серьезным препятствием.

Зависимость от teacher model. DCD требует предобученной SASRec модели. Качество distillation ограничено ceiling'ом teacher'а. Если SASRec на конкретном домене работает плохо, DCD не сможет компенсировать дефицит collaborative signal.

Чувствительность annealing schedule. Результаты по collision rate показывают, что неправильный выбор temperature schedule может свести на нет все преимущества. Слишком быстрое hardening ведет к collapse, слишком медленное -- к train-inference mismatch. В статье не дается автоматического способа выбора schedule.

Масштаб экспериментов. Максимальный размер каталога -- 33K items (Upwork). Для industrial-scale систем с миллионами item'ов поведение soft identifiers не протестировано. Скорее всего, проблемы с collapse и collision будут усиливаться.

Нет online A/B тестирования. Несмотря на наличие Upwork в авторах, статья не содержит online deployment results. Это ограничивает понимание того, как end-to-end обучение ведет себя в production-цикле с обновлением каталога.

Identifier drift. End-to-end updates меняют коды item'ов в процессе обучения. В production это требует versioning и контроля drift'а -- темы, которые статья не рассматривает.

15. Практические выводы

Soft identifiers -- мощный, но капризный инструмент. Gradient bridge через soft identifiers действительно позволяет recommendation loss влиять на tokenizer. Но без трех защитных механизмов (AIA, CUR, DCD) этот мост быстро превращается в источник нестабильности.

Annealing -- ключевой элемент. При реализации подобного подхода temperature annealing должен быть настроен в первую очередь. Рекомендуется логировать collision rate, code entropy и recommendation metrics совместно на каждом этапе.

CUR как анти-collapse baseline. Uniformity regularization -- простой и эффективный механизм, который можно использовать и в других подходах к semantic tokenization, не обязательно в контексте soft identifiers.

Distillation от lightweight teacher'а. Идея DCD может быть перенесена в другие generative recommendation системы. Вместо тяжелого LLM-teacher'а достаточно простой collaborative filtering модели (SASRec, GRU4Rec) для передачи interaction patterns.

After annealing -- обязательная проверка. После окончания joint training необходимо провести evaluation в hard-ID режиме (без soft path), чтобы убедиться, что inference mode действительно работает.

16. Связь с соседними работами

UniGRec близок к DIGER: обе работы делают tokenizer дифференцируемым, но DIGER фокусируется на Gumbel exploration против collapse, а UniGRec -- на soft identifiers, annealing и distillation. С PIT его роднит co-optimization, но PIT добавляет dynamic beam index для evolving mappings. С ETEG-Rec общий мотив -- преодоление decoupled tokenizer'а, но ETEG-Rec использует alternating optimization вместо true end-to-end. С ReSID связь в критике decoupled tokenizer'а, но ReSID не требует LLM/end-to-end soft identifiers.

В более широком контексте UniGRec можно рассматривать как ответ на вопрос, поставленный CoST: если tokenization objective так важна (CoST заменяет MSE на contrastive), то не лучше ли оптимизировать tokenizer напрямую под recommendation loss? UniGRec отвечает: да, но нужны три защитных механизма.

17. Итоговая оценка

UniGRec -- одна из наиболее прямых и систематических попыток сделать semantic ID частью единой recommendation objective. Статья ценна не столько конкретными архитектурными решениями, сколько четкой диагностикой трех failure modes end-to-end обучения и предложением решений для каждого из них. Ablation study убедительно показывает вклад каждого компонента.

Главный практический вывод: soft identifiers дают градиентный мост между tokenizer'ом и recommender'ом, но без annealing, uniformity regularization и collaborative distillation этот мост быстро превращается в источник mismatch'а, collapse'а или потери collaborative signal. Это делает UniGRec важным reference point для будущих работ по joint optimization в generative recommendation.

18. Что читать после UniGRec

  1. TIGER -- базовый framework, чтобы понять staged training baseline.
  2. EAGER -- behavior-semantic collaboration, альтернативный подход к обогащению tokenizer'а.
  3. LETTER -- learnable item tokenization, alternating optimization.
  4. CoST -- contrastive tokenization, другой подход к retrieval-aware tokenizer loss.
  5. DIGER -- differentiable semantic ID через Gumbel-Softmax, ближайший конкурент по идее end-to-end обучения.
