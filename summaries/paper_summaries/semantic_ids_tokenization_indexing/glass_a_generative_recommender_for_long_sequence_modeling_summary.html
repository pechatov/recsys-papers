GLASS: A Generative Recommender for Long-sequence Modeling via SID-Tier and Semantic Search

← К общему списку статей

Авторы: Shiteng Cao, Junda She, Ji Liu, Bin Zeng, Chengcheng Guo, Kuo Cai, Qiang Luo, Ruiming Tang, Han Li, Kun Gai, Zhiheng Li, Cheng Yang.

Аффилиации: Kuaishou Inc.; Tsinghua University; BUPT.

Коротко

GLASS улучшает generative recommender для длинных пользовательских последовательностей. Два основных блока: SID-Tier, который учитывает tiered/prefix nature semantic IDs, и Semantic Hard Search, который помогает decoding и mid/late token prediction.

Контекст

TIGER-подобные модели страдают от длинных histories: sequence length растет в токенах, а ошибка на раннем SID token ломает весь item. При этом первый токен semantic ID часто задает coarse bucket, а последующие уточняют item. Стандартный Transformer не всегда явно использует эту иерархию.

Проблема

Для long-sequence recommendation нужно одновременно сохранить long-term interest и точно генерировать SID. Наивная генерация всех токенов равноправно недооценивает prefix dependency: если coarse token неверен, следующие почти бесполезны. Также возникает sparsity: редкие items плохо представлены в SID sequence training.

Метод / архитектура

SID-Tier перестраивает modeling around SID levels: coarse-to-fine tokens получают специальное внимание и gating. Semantic Hard Search добавляет semantic retrieval/search на этапе генерации, чтобы использовать близость в item embedding/SID space. Sparsity-aware augmentation включает codebook resizing и semantic neighbor augmentation для редких регионов.

Objective / алгоритм

Модель обучается как SID-based generative recommender, но inference дополняется semantic search. Алгоритм декодирует prefix, оценивает gate/sequence length, затем применяет hard search среди семантически допустимых candidates. Это снижает зависимость от чистого autoregressive beam и помогает восстановить кандидатов, если поздние SID-токены менее уверенные.

Эксперименты и метрики

Датасеты: Taobao-MM и KuaiRec. Метрики: Hit@K и NDCG@K. В ablation авторы показывают, что SIDTier поднимает Recall@5 примерно с 0.0702 до 0.0842, а совместный Tiger+SIDTier+SHS улучшает Recall@10/NDCG@10. Codebook resizing дает пик Recall@10 около 0.13 в описанном эксперименте. Также анализируется average gate value vs retrieved sequence length.

Сильные стороны

  • Метод явно использует иерархию semantic IDs.
  • Комбинирует генерацию с semantic search, что практичнее для long sequence.
  • Есть анализ gating и sparsity augmentation.

Ограничения

Сложность системы выше, чем у чистого TIGER: появляется search component и дополнительные hyperparameters. Результаты завязаны на качество SID tokenizer и на доступность semantic neighbor structure. Online A/B в исходнике не является основной раскрытой частью.

Как реализовать / проверять

Начать с baseline TIGER на тех же SIDs, затем добавить SID-level attention/gating и только после этого Semantic Hard Search. Нужно логировать accuracy по позициям SID, prefix ceiling, gate value, Recall/NDCG и latency search. Для tail-items отдельно проверять effect codebook resizing и neighbor augmentation.

Связь с соседними работами

GLASS дополняет variable-length SID и Spotify GLIDE: все три работают с token budget/длинными последовательностями, но GLASS фокусируется на tier-aware generation. С RecoChain его роднит идея добавить post-generation search/ranking вместо чистой генерации.

SID-Tier: зачем нужна tier-aware модель

Semantic ID обычно имеет coarse-to-fine структуру.

Первый token выбирает крупный semantic cluster.

Следующие tokens уточняют subcluster и конкретный item.

Vanilla Transformer видит эти tokens как обычную последовательность.

SID-Tier вводит inductive bias: уровни SID не равноправны.

Если ошибка возникает на первом уровне, дальнейшее уточнение почти не спасает item retrieval.

GLASS анализирует accuracy и probability по уровням: acc_1, P_1, acc_2, P_2.

Semantic Hard Search

SHS дополняет autoregressive decoding semantic search.

Если prefix указывает на semantic region, не обязательно полагаться только на следующий generated token.

Можно использовать hard search по semantic neighbors и valid candidates.

Это особенно полезно для mid-to-late stages sequence generation.

В ablation авторы отмечают, что Tiger+SIDTier+SHS поднимает acc_2 и P_2.

Результат - лучшая Recall@10 и NDCG@10.

Sparsity-aware augmentation

Long-tail items плохо обучаются, потому что у них мало behavioral signal.

GLASS использует Codebook Resizing и Semantic Neighbor Augmentation.

Codebook Resizing меняет емкость discrete space, чтобы уменьшить перегрузку bucket'ов.

Semantic Neighbor Augmentation добавляет соседние items как дополнительные training signals.

В ablation CR дает сильный прирост ranking capability.

Комбинация SIDTier+SHS+CR достигает peak Recall@10 около 0.13 в описании авторов.

Gating mechanism и setup

Gate регулирует, сколько токенов или какой retrieved sequence length использовать во время inference.

Figure с average gate value сравнивает configurations [128,128,128] и [64,128,128].

Если gate слишком мал, модель возвращает coarse candidates.

Если gate слишком велик, latency растет и ошибки поздних tokens могут ухудшить precision.

Taobao-MM и KuaiRec используются как основные датасеты.

Метрики Hit@K и NDCG@K отражают top-k retrieval/ranking quality.

Failure modes и проверка

Первый failure mode - неверный coarse SID token.

Второй - semantic search возвращает близкий, но уже просмотренный или недоступный item.

Третий - neighbor augmentation усиливает popularity bias, если neighbors берутся из head clusters.

Четвертый - codebook resizing может улучшить Recall@10, но ухудшить stability SID vocabulary.

Пятый - gate miscalibration на редких histories.

  1. Считать per-level SID prediction accuracy.
  2. Добавить prefix-validity constraints в beam/search.
  3. Построить semantic neighbor index offline.
  4. Сравнить vanilla TIGER, TIGER+SIDTier, TIGER+SHS и полный GLASS.
  5. Отдельно оценивать head/tail и short/long histories.
  6. Логировать latency semantic search и долю fallback cases.

Итог

GLASS показывает, что semantic ID - это не плоская строка токенов. Если model и decoding учитывают coarse-to-fine структуру, long-sequence generative recommendation становится устойчивее и точнее.
